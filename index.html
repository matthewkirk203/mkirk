Electrical Engineering

Electrical Engineering is a fascinating field. It opens so many doors to so many different projects. I have found a lot of interest in the aerospace industry and have always had a personal fascination with that which lies above us. More specifically, I’ve enjoyed the controls classes as well as the signal processing and digital communication classes I’ve taken.

As far as control experience goes, I've taken an intro to controls class where we model the dynamics of a system and then apply different control methodologies to it. The control methodologies we used were PID, Observer based control, and Loopshaping control. This led me into taking a grad level class as an undergrad where we study flight dynamics and control. We primarily control the aircraft with PID and use a Kalman filter to help with state estimation.

I also have a good bit of experience programming. I've worked as a Computer Science TA using C++; I've taken a couple of classes where I've had to use C for embedded programming; and I currently maintain and improve the website/database for my job using the standard HTML, CSS, Javascript, PHP, MYSQL.

The McKay School of Education is one of the colleges on BYU campus. I spent a lot of time working as a CSR (Computer Support Representative).

The primary responsibility is supporting faculty and staff when they have issues with computers and technology; this can be anything from "how do I print" to "my computer won't turn on". Naturally, this lends itself to a lot of debugging and researching problems. Sometimes we can fix the problem, but sometimes we have to consult the University IT department, as we technically exist under them, so the problem may be with one of their services. If it's a hardware problem, we fix it if we can, but we will also ship things back to the manufacturer if it is under warranty.

In addition to helping faculty directly, the secondary objective is to improve our work environment by automating processes with deployment tools, scripts, etc, as well as maintaining servers, keeping track of inventory/users/licenses/etc. in our database, among other more granular things.

Personally, I've done just about everything. I started as being in charge of security, which largely involved me changing passwords on dozens of servers as well as patching security holes caused by outdated software and packages. I also managed group security permissions in Active Directory so people would only be able to log into certain computers or have access to certain folders on the server.

I also worked a lot on maintaining both the front end and back end of our database. We have a PHP and MYSQL back end and use HTML, CSS, Javascript, and jQuery for the front end. I had to figure out much of it by myself as I previously had little experience in anything web development, the code was almost entirely uncommented, and the guy that built most of it was leaving. I made several improvements and fixed bugs as they showed up.

I've also spent a good amount of time working with deployment tools such as Casper and SCCM as well as scripting (I prefer bash scripting to batch). That involved setting up policies to install software for faculty or lab machines, or to perform various maintenance such as cleaning up files from lab machines on a scheduled basis.

Right now I also maintain our servers. I schedule restarts, take snapshots, and apply patches when they need it. We have Windows, Mac, and Linux servers, but we have mostly Linux servers. But I don't only do that. I still write scripts and I still maintain the database when it needs it. Since I will be leaving in April, I've entered more of a training role. I help the newer employees get acquainted with our systems and try and capture the things I've learned into our wiki.
